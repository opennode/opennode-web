## Project Overview

OpenNode corporate website (opennodecloud.com) — a static site built with Hugo. Single-page marketing site for OpenNode, the company behind the Waldur open-source cloud management platform.

## Development Commands

```bash
# Local dev server
hugo server

# Build for production (used in CI)
hugo --minify
```

Requires Hugo Extended edition.

## Architecture

- **Hugo static site** with no theme — custom layouts only
- **Single page**: all content is in `layouts/index.html` using sections (hero, waldur, services, about, contact)
- **Base template**: `layouts/_default/baseof.html` — loads Google Fonts (JetBrains Mono), processes CSS via Hugo Pipes (minify + fingerprint)
- **Partials**: `layouts/partials/header.html` (fixed nav + mobile menu toggle JS), `layouts/partials/footer.html`
- **Styling**: Single CSS file at `assets/css/main.css` — dark terminal/hacker aesthetic with CSS custom properties in `:root`
- **Config**: `hugo.toml` — site params (description, email, waldurURL) referenced in templates via `.Site.Params`
- **No JS dependencies** — only inline vanilla JS for mobile menu toggle

## Deployment

GitHub Actions (`.github/workflows/deploy.yml`) deploys to GitHub Pages on push to `main`. The `static/CNAME` file maps to opennodecloud.com.

## Key Conventions

- Design uses a dark theme with monospace typography (JetBrains Mono) and terminal-window visual motifs
- Color palette defined as CSS custom properties: `--color-bg`, `--color-primary` (teal #0d7377), `--color-green` (#4ade80)
- The `public/` directory is committed but is a build artifact — regenerated by `hugo --minify`
